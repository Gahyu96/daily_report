# 设计文档：日报格式优化 + 日历集成

**日期**: 2026-03-27
**状态**: 已批准

---

## 背景

当前日报可读性较差，缺乏时间维度组织，且飞书日历数据虽已采集但未集成到报告内容中。周报和月报格式也需要同步优化。

---

## 目标

1. 日报按上下午时间段组织内容，提升可读性
2. 修复飞书日历数据缺失问题，用于驱动时间分组和会议信息补充
3. 周报/月报加亮点卡片，格式更清晰
4. 周报和月报都走 LLM 生成（月报目前是 JSON mock，需要修复）

---

## 改动范围

| 文件 | 改动内容 |
|------|---------|
| `daily_report.py` | `build_combined_text` 加入 `feishu_calendar` |
| `generator.py` | 重写 `DAILY_PROMPT_PREFIX` + `_build_daily_prompt`；重写 `_build_weekly_prompt`；新增 `_build_monthly_prompt` + `_call_llm_for_monthly`（替换 JSON mock） |

---

## 详细设计

### 1. 修复日历数据集成（`daily_report.py`）

**根因**：`build_combined_text` 函数未处理 `feishu_calendar` 键，导致日历数据采集后被丢弃，从未传入 LLM。

**修复**：在 `build_combined_text` 的 `parts` 拼接中加入：

```python
if structured_data.get("feishu_calendar"):
    parts.append("=== 飞书日程 ===\n" + structured_data["feishu_calendar"])
```

### 2. 日报格式（`generator.py`）

#### 2.1 新增日历数据使用说明（加入 `DAILY_PROMPT_PREFIX`）

```
【重要：飞书日程说明】
输入中可能包含「飞书日程」部分，这是当天的日历事件数据。请：
1. 用日程的开始时间判断事件属于上午（12点前）还是下午/晚上（12点后）
2. 将会议类日程与飞书聊天/纪要对照，补充完整的会议信息
3. 日程时间作为上下午分组的主要依据；无日程时，根据消息时间戳推断
```

#### 2.2 日报输出格式

```markdown
# 日报 - YYYY-MM-DD

## 一、今日概览
[100-300 字整体总结，说明上下午各自的工作重心]

## 二、核心工作内容

### 🌅 上午（HH:MM-12:00）

**🎯 会议**
- 事项标题
  - 关键子点 1
  - 关键子点 2

> 📎 来源: 飞书智能纪要 xxx.docx · 飞书会话 xxx群

**💻 自主工作**
- 事项标题
  - 子点

> 📎 来源: Claude 项目会话

### 🌆 下午（13:00-HH:MM）

**👥 团队管理**
- ...

> 📎 来源: ...

**🤝 提供支持**
- ...

> 📎 来源: ...

### 🌙 晚上（18:00以后）（如有内容才显示）
...

## 三、问题与风险
- 问题描述

## 四、明日计划
- [ ] 任务内容 - 时间节点

## 五、需要支持
- 谁: 需要什么支持

## 六、其他备注
【可能计划执行】的任务列表

---

## 附录：数据源索引
- Claude 历史会话: N 条
- Claude 项目会话: N 个
- 飞书会话: N 个
- 飞书文档: N 个（含智能纪要 M 个）
- 飞书日程: N 个事件
```

**规则**：
- 没有内容的时间段不显示
- 某时间段内没有某类工作不显示空章节
- 时间段括号内的时间根据日历事件动态调整
- 每个事项标题下用子弹点展开 2-3 个关键细节
- 多个来源用 `·` 分隔在同一 `> 📎 来源:` 行
