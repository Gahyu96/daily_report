# 日报增强功能设计文档

**日期**: 2026-04-16
**版本**: v1.0
**作者**: AI Agent

---

## 一、背景与目标

### 1.1 需求背景
用户希望在现有日报结构基础上，新增三个内容模块：
1. 今天的金句
2. 今天可以总结到的决策逻辑和SOP方法论
3. 今天的行业资讯

### 1.2 目标
- 增强日报的知识沉淀价值
- 从记录型日报升级为学习型日报
- 帮助用户沉淀可复用的方法论和洞察

---

## 二、设计方案

### 2.1 章节位置
在现有日报结构中，新增章节位于：
- **之前**: "六、其他备注"
- **新增**: "七、今日金句"、"八、决策逻辑与SOP方法论"、"九、行业资讯"
- **之后**: "附录：数据源索引"

### 2.2 章节详细设计

#### 七、今日金句
```markdown
## 七、今日金句
> [1-2句有启发性的话，来自今日工作中的感悟、会议中的发言、或读过的资料]
> 来源: [相关来源]
```

**Prompt 要求**:
- 从今日工作内容中提炼 1-2 句最有启发性的话
- 可以是会议中的精彩发言、自己的感悟、或从文档中读到的关键点
- 如无合适内容，此章节可省略

---

#### 八、决策逻辑与SOP方法论
```markdown
## 八、决策逻辑与SOP方法论

### 决策逻辑
- [决策1名称]
  - 背景: [为什么需要做这个决策]
  - 选项: [考虑过哪些方案]
  - 结论: [最终选择及核心理由]
  - 来源: [相关来源]

### SOP沉淀
- [SOP名称]
  - 适用场景: [什么时候用这个SOP]
  - 操作步骤: [1/2/3...]
  - 注意事项: [关键风险或避坑点]
  - 来源: [相关来源]
```

**Prompt 要求**:
- **决策逻辑**: 记录今天做过的重要决策，用"背景-选项-结论"三段式呈现
- **SOP沉淀**: 如果今天摸索出了可复用的工作流程，总结成标准化操作步骤
- 如无合适内容，此章节或其子章节可省略

---

#### 九、行业资讯
```markdown
## 九、行业资讯
- [资讯标题]
  - 核心要点: [2-3个关键信息]
  - 对我的影响/启发: [这个资讯与我工作的关联]
  - 来源: [相关来源]
```

**Prompt 要求**:
- 如果今天关注到行业动态、技术趋势、竞品信息，记录下来并思考对自己的影响
- 如无合适内容，此章节可省略

## 三、实施范围

### 3.1 需要修改的文件

| 文件 | 修改内容 |
|------|---------|
| `generator.py` | 更新 `DAILY_PROMPT_PREFIX`、`_build_daily_prompt()`、fallback 模板、空报告模板、mock 结果 |

### 3.2 不需要修改的文件
- 周报、月报相关代码（本次仅增强日报）
- collector.py、cache_manager.py 等数据收集模块
- feishu/ 目录下的飞书集成模块

---

## 四、验证标准

1. **Prompt 更新**: `_build_daily_prompt()` 输出的模板包含新章节
2. **Fallback 兼容**: `_generate_fallback_report()` 和 `_write_empty_daily()` 包含新章节框架
3. **Mock 数据**: `_get_mock_daily_result()` 包含新章节示例
4. **向后兼容**: 旧日报格式不受影响，新日报仅新增内容

---

## 五、风险与预案

| 风险 | 影响 | 预案 |
|------|------|------|
| LLM 不按新模板输出 | 新章节缺失 | 在 prompt 中强化格式要求，保留 fallback 机制 |
| 新增内容质量不高 | 日报可读性下降 | 如 LLM 生成内容质量差，可省略相关章节 |
| prompt 过长导致 token 超限 | 调用失败 | 控制新增 prompt 长度，优化表述 |
